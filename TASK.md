Challenge: Automated Content Briefing System

Objective
Design and build a system that takes a long-form piece of content (text) and converts it into a
concise, structured multimedia briefing.

Core Constraint (IMPORTANT)
We are evaluating your ability to:
- Use open-source tools and models (LTX-2.3, Wan 2.2, etc)
- Design cost-efficient pipelines
- Make practical engineering tradeoffs

Input
Choose any topic (e.g., a technical concept, news article, or report)
Input format: text or PDF

Output
Produce a 3–5 minute “briefing-style” output that:
- Clearly communicates the key ideas
- Is structured into sections (intro, key points, summary)
- Includes both:
- Visual elements (slides, images, or generated visuals)
- Audio narration or explanation

Requirements
Your system should demonstrate:
- Content processing: Extracting and structuring key ideas
- Transformation pipeline: Converting content into a structured format
- Multimedia generation: Combining visuals and audio

Restrictions
- Do not rely primarily on paid APIs (e.g., OpenAI, etc.)
- If used, they must be minimal and clearly justified

Deliverables
Please submit:
- Final output (video or multimedia format)
- Code repository (GitHub preferred)
- Brief explanation (1–2 pages) covering:
- System architecture
- Tools/models used
- Cost considerations

Time Expectation
48 hours (focus on approach and clarity, not perfection)
Submission Instructions

Reply with:
- GitHub repository link
- Output file or link (Google Drive, YouTube unlisted, etc.)
- Explanation document or notes

Evaluation Criteria
- System design and structure
- Cost-efficiency and engineering decisions
- Clarity and usability of the output
- Creativity in solving the problem
